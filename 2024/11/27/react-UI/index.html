React 描述UI

1、组件构成UI

React中允许将标签、css、js组合形成组件，即可以复用的UI元素
定义组件 注意组件名要开头大写！

定义后在其他组件中使用

组件的定义不能嵌套，应该全部定义完后再在其他组件中使用

2、组件的导入与导出

默认导出和默认导入

默认导出的话组件名不一定要与上个文件相同
一般如果只有一个组件的话使用默认导出

具名导出和具名导入
具名导出时只需要用export，省略default
具名导入时需要加{},并且组件名必须相同

一般要导出多个组件时使用具名

3、JSX规则

  • 1、只能返回一个根元素
    如果一个组件中包含多个元素，需要用一个父标签把他们包裹起来

    原因：JSX在底层被转化为JavaScript对象，在一个函数中只能返回一个对象

  • 2、标签必须闭合

<img>这样的自闭合标签必须写成<img/>

  • 3、使用驼峰式命名法给大部分属性命名
    eg:css中背景颜色用background-color表示，在JSX中只能用backgroundColor(此处的规则与JavaScript相同)
  • 4、JSX中的大括号
    用{ }将js变量引入，例如 只能用作JSX标签内的文本和跟在=后面的属性
    <{tag}>Gregorio Y. Zara’s To Do List</{tag}>
    标签名不能用
    src=”{avatar}”输出的是{avatar}而不是avatar变量的值

5、将JSX作为子组件传递

子组件的 props.children 可用于在调用该子组件时访问开始和结束标记之间的嵌套值。

用props传递属性和JSX的区别：第一种只能改变标签内的内容，而用children可以传递html的框架

key值的重要性
保持列表项的顺序，类似于html中的className，是一个标识告诉react每个组件对应数组中的哪一项。key值应该在初始数据中就存在而不可以在渲染过程中随意更改，并且在同一个数组中key值不能重复，不同数组中不做要求

7、纯函数

纯函数的特点：
1、只负责自己的任务，不会更改在函数调用前产生的变量或对象
2、输入相同则输出相同
错误示例：不符合纯函数，更改了预先存在变量的值，也叫做“突变”

改为正确的写法:用props传递

此时的组件是纯粹的，没有外部变量的影响，每次调用只计算自己的JSX
只有事件处理函数不需要是纯函数，因为不管是否在组件内定义都不会在渲染时执行

8、UI具有树状结构

树结构主要分为两种：渲染树和模块依赖树
渲染树主要是以组件划分
模块依赖树主要是将每个引入的子文件视为模块，再以模块进行划分，每个分支代表import语句
树状结构主要是用于使react整体的代码结构化
