react组件重渲：

引起重复渲染的原因：

在父组件更新状态重新渲染时子组件也会重新渲染

1.通过拆分组件阻止重复渲染

在一个屏幕中可能需要创建很多个状态，但是有些状态是关联的，有一些是独立的，可以将一部分关联的状态拆分成组件，防止于其无关的部分每次重新渲染，导致页面加载十分缓慢

！！！不要在组件中创建组件，否则每次状态更新，组件中的组件每次都会销毁重新创建，比如输入框TextInput每次输入一个字都会失焦

1.如果一部分使用state，另一部分和state相对独立，那么将独立的单独设为一个组件，避免重渲

2.如果有耗时很长且不涉及状态的部分，将含状态管理的部分拆分成组件，剩下的作为children props

2.对于父组件导致子组件重新渲染：

在 React 中，当子组件的状态更新时，React 会根据组件的依赖关系和更新机制来决定是否重新渲染父组件。如果传递给子组件的是一个固定的值，而不是与父组件状态相关的变量，且子组件没有其他依赖于父组件状态的逻辑，那么子组件重新渲染不会导致父组件重新渲染。

所以在这种情况下可以拆分子组件

此时点赞就不会重新渲染一遍评论，导致加载速度非常慢

如果父组件向子组件传递的不是固定值，而是状态量，那么不论传递的状态是否改变，子组件都会跟随父组件重新渲染，因为父组件的状态全部改变了，

那么要避免子组件重渲则需要使用React.memo优化

React.memo

它通过记忆组件的渲染输出，在组件的 props 没有变化时跳过重新渲染，从而提高性能

1.基本使用：将一个函数组件作为参数传递给React.memo，并返回一个记忆化的组件。

记忆化就是将传递给子组件的props缓存，与变化后进行对比，如果没有改变则不重新渲染，如果发生改变了就重渲

这样的机制就很适用于子组件接收父组件的状态值，在必要时才重新渲染优化性能

React.memo还可以自定义子组件何时渲染，这时需要在React.memo中传递第二个自定义比较函数，用于控制渲染时机

但是如果想通过子组件改变父组件的值就需要传递函数(状态提升)，这时想要缓存函数就要用到useCallback

useCallback是用来缓存函数，有两个参数，一个是缓存函数，一个是依赖项。如果依赖项没有改变，会直接返回相应的缓存结果，避免子组件的重新渲染

如果此时的allPlus中的set是直接赋值而不是以函数的形式，那么缓存函数则会获取不到最新的值

useState使用直接传惨和回调函数传参的区别是直接传参是传递一个状态，而回调函数是传递在上一个状态基础上的操作

useCallback和useMemo的区别：

useCallback缓存的是函数，而useMemo缓存的是计算的值。

useMemo中传递的是一个不含任何参数的纯函数以及依赖项，如果依赖项不改变直接返回计算的值

此处再提一下useRef：

useRef操作dom元素的操作先不谈，useRef可以用来缓存值，但是不引发渲染，起到一个记忆的作用。

同样是缓存值，那么useRef和useMemo有什么区别？

useRef是不管值是否改变都会缓存记录，而useMemo是记录值是否改变减少重复计算提升效率
